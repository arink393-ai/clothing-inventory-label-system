帳號或密碼錯誤

管理人員可使用全部功能。店鋪人員密碼為自己的員工編號（如 E-001），登入後結帳會自動記錄是哪位員工；未建檔者可用通用密碼。

設定議價加成與編號規則

每件商品依分類自動編號（例：雜貨＝MSC-0001）。建議售價會在你輸入的定價上加成，方便客人議價。

舉例：定價 350 + 加成 15% → 進位至 10 → 建議售價 410
⚠ 這個瀏覽器無法自動儲存（可能是無痕模式或權限限制），離開後資料會消失。請改用一般視窗開啟，並用「備份」匯出檔案保存。

匯入從 Excel 批次建檔

Excel 需要欄位：商品名稱、分類、定價（可選 建議售價）。系統會自動編號並產生條碼。

組合A＋B 組合價

設定搭售優惠價。A、B 可各自選「分類」或「單一商品」（例：分類 T恤 ＋ 分類 雜貨）。結帳時購物車湊齊一組就以組合價計（多組多算）；同品項量販／第二件／買送活動則在各商品的「單品活動」設定。組合價需低於原價總和，否則無法新增。

分類商品分類管理

可自由新增分類。每個分類有一組「編號前綴」（英數），用來自動產生商品編號（例：MSC-0001）。前綴可留空，系統會自動配一組。

清單商品總覽

庫存會在「進貨入庫」與「銷售結帳」時自動加減。手動盤點時可逐列改數字後按「存」，或一次改多筆後按上方「一鍵儲存庫存」全部存檔（按鈕變綠代表該列尚未存）。

資料備份 / 還原 / 清空（全部資料）

這裡處理的是整套資料：商品、進貨、銷售、退貨、會員與點數、各項設定全部包含在內。資料只存在本機，換手機或換瀏覽器請務必先備份再到新環境還原。

尚未備份

「備份」會下載一個 .json 檔；「還原」會用該檔覆蓋目前全部資料；「清空全部資料」會把所有資料刪除歸零（含會員），請務必先備份。

盤點庫存盤點（實際 vs 系統）

實際清點後在此輸入各品項的「實際數量」，系統會算出差異；按「套用盤點」會把庫存更新為實際數量，並留下一筆盤點紀錄。有變體的商品會逐規格列出。

安全登入密碼設定

管理員可用帳號 admin（或 SHIAWASE_ZAKKA）登入；店鋪人員可用帳號 staff 登入，密碼為下方的店鋪人員密碼（預設 1234）或自己的員工編號。可在此自訂兩組共用密碼。

變更後立即生效，下次登入請使用新密碼。密碼會隨「備份」一起保存。

AI拍照辨識設定（Gemini）

設定後，新增商品時可用「📷 拍照命名」自動辨識商品、帶入名稱與介紹。需要一組 Google Gemini 免費 API 金鑰（在 Google AI Studio 免費申請）。金鑰只存在本機瀏覽器，會隨備份保存。

申請網址：aistudio.google.com/app/apikey（登入 Google 帳號 → Create API key）。免費額度足夠一般建檔使用。

同步多裝置雲端同步（手機／電腦共用）

設定後，手機與電腦會透過雲端資料庫共用同一份資料——手機建檔、電腦結帳都看得到。需要一組免費 Supabase 專案。狀態：未連線

每台裝置都要填一次同樣的設定。金鑰與密碼只存在本機、不會寫進網頁程式，也不會同步到雲端。建立資料表的步驟我會另外提供。
